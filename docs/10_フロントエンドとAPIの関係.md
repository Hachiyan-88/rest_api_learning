# フロントエンドとAPIの関係

## はじめに

これまでのページで、REST・HTTP・JSON等の、API通信の仕組みについて整理してきました。

その道中にふと
>「APIって結局、どこで使われているの？」  
>「自分の見ているWEB画面と、どう関係しているの？」

このページでは、フロントエンド（画面）とAPIの関係を整理し、  
WEBアプリがどんな感じでデータを取得・表示しているのかを理解するためにまとめたものです。

ゴールとしては  
**「画面の裏側で何が起きているのか」**
をイメージできるようになることとします。

---

## フロントエンドとバックエンドの役割

### フロントエンド（画面側）

ユーザーが直接触れる部分のこと。

- 画面表示
- ボタン操作
- 入力受付
- データの表示

### バックエンド（サーバー側）

見えないところで処理を担当する部分のこと。

- データの保存
- データの処理
- 認証
- ビジネスロック

---

## APIは両者をつなぐ橋

👉フロントとサーバーは直接会話しない
👉 APIを通してデータをやり取りする

```CSS
[フロントエンド]
        ↓
    API通信
        ↓
[バックエンド]
```

---

## 実際の動き：タイムライン表示

例：SNSアプリを開いた時

### ①画面を開く

ユーザー操作
↓

### ②フロントがAPIへリクエスト

```bash
GET /posts
```

↓

### ③サーバーがJSONを返す

```json
[
    {"user":"A","text":"こんにちは"},
    {"user":"B","text":"今日は暑い"}
]
```

↓

### ④フロントが画面に表示させる

📱 タイムライン完成

---

## 図で理解する通信の流れ

```javascript
ユーザーが画面を開く
        ↓
フロントエンドがAPIへリクエスト
        ↓
サーバーがデータ（JSON）を返す
        ↓
フロントエンドが画面に表示
```

👉画面はデータを持っているわけではない  
👉必要なときに取得している

---

## ログイン処理の流れ

### ①ログインボタンを押す

フロント→

```bash
POST /login
```

```json
{
    "email":"user@email.com",
    "password":"1234"
}
```

### ②サーバーが認証する

✅正しければOK  
✅間違いならエラー

### ③レスポンス

```json
{
    "token":"abc123"
}
```

### ④フロントが保存する

👉トークン保存  
👉ログイン状態を維持する

---

## フロントエンドがAPIを使う場面って？

### データ取得

```bash
GET /posts
```

### 新規作成

```bash
POST /posts
```

### 更新

```bash
PATCH /users/1
```

### 削除

```bash
DELETE /posts/10
```

👉これらはすべて画面操作に対応している

---

## POSTMANでの操作との関係

今行っているAPIテストは：

👉フロントエンドの代わりに  
👉手動でAPI通信をしている状態

つまり、アプリの裏側の通信を直接体験している。

### 📒Note

APIは通常、プログラムから呼び出されるが、  
**Postman**というツールを使うと画面操作だけでAPI通信を試すことができる。

ここでも使用しているため、  
導入方法と基本操作を次のページで簡単に整理しておきます。

👉 詳しくは → [11_Postman入門](./11_Postman入門.md)

## 日常に例えてみると？

### 🛒ネットショップ

商品一覧ページを開く  
↓  
フロント：

```bash
GET /products
```

↓  
サーバー：  
商品データ（JSON）  
↓  
画面：  
📺商品一覧が表示される

---

### カートに追加

ボタンを押す  
↓  

```bash
POST /cart
```

↓  
商品がカートに入る

---

👉私たちが押しているボタンの裏側では  
👉API通信が行われている。

---

## なんでこの仕組みが重要なの？

もしAPIがなかったら・・・

- ページ更新のたびに再読み込みが必要
- モバイルアプリとデータ共有できない
- リアルタイム表示が難しい

👉現代のアプリは成立しない

## まとめ

- フロントエンドは画面表示を担当する
- バックエンドはデータ管理と処理を行う
- APIは両者をつなぐ通信手段
- 画面の表示はAPIから取得したデータで作られている
- POSTMANでの操作は、アプリの裏側通信の体験そのもの

---

### 自分用メモ

- 画面はデータを持っているわけではない
- 必要なときにAPIから取得している
- ボタン操作の裏側でHTTP通信が行われる
